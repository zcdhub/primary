第一天 0.概述 B/S 浏览器服务器框架 优点： 1.分布性强，客户端零维护； 2.业务扩展方便，通过增加网页即可增加服务器功能； 3.维护简单方便，改变网页即可实现用户的同步更新； 4.开发简单，共享性强 缺点： 1.个性化特点明显，无法实现具有个性化的功能需求； 2.跨服器上，bs架构不尽人意； 3.客户端->服务器的交互方式是请求->相应，通常动态刷新界面，相应速度明显降低； 4.在速度和安全性能上花费成本较高； 5.功能弱化，难以实现传统模式下的特殊功能要求； C/S 客户端服务器框架 优点： 1.能充分发挥pc端的处理能力，很多工作可以在PC端处理后再发送到服务器，所以cs客户端响应速度快； 2.操作界面漂亮、形式多样、可以充分满足客户端自身的个性化要求； 3.C/S结构的管理信息系统具有较强的事务处理能力，能实现复杂的业务流程。 4.安全性能可以很容易保证。 缺点： 1.需要专门的客户端安装程序，分布功能弱； 2.兼容性差，对于不同的开发工具，具有较大的局限性； 3.开发、维护成本较高； 4.用户群固定； html 超文本标记语言 CSS 层叠样式表 JS(JavaScript) 动态效果制作的直译式脚本语言 标签：组成网页的最小单位(规定的) 双:<标签名 属性1="值1"...> 单:<标签名 属性1="值1".../> 1.全局架构标签: 设置字符集： 另：php中设置字符集 header("Content-Type:text/html;charse=utf-8") 2.body标签属性 3.实体字符集   不间断空格 ( )   全角空格(宽度正好为一个字符) & 连接符 (&) > 大于号 (>) < 小于号 (<) ' 单引号 (') " 双引号 (") ® 注册商标符 ('圈R') © 版权符 ('圈C') ™ 商标符 (TM) × 乘号(x) ÷ 除号 4.常用字体修饰标签 ； 均为加粗 ； ；均为斜体 下划线 删除线 下标签 上标签 5.段落
